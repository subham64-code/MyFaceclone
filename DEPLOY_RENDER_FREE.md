# Deploy MyFaceclone on Render (Free)

This project already includes backend + frontend (Django templates/static files), so one free Render Web Service is enough.

## 1) Push this code to GitHub
- Create a GitHub repository.
- Push this project to that repository.

## 2) Create service from Blueprint
- In Render dashboard: New > Blueprint.
- Connect your GitHub repo.
- Render will detect `render.yaml` and create the service automatically.

## 3) Environment used automatically
- `DJANGO_SECRET_KEY` is auto-generated by Render.
- `DJANGO_DEBUG=False`.
- `DJANGO_USE_MANIFEST_STATIC=True`.
- `DJANGO_ALLOWED_HOSTS=.onrender.com`.
- `DJANGO_CSRF_TRUSTED_ORIGINS=https://*.onrender.com`.

## 4) Build and start commands
Already configured in `render.yaml`:
- Build: `pip install -r requirements.txt && python manage.py migrate && python manage.py collectstatic --noinput`
- Start: `gunicorn MyFaceclone.asgi:application -k uvicorn.workers.UvicornWorker --bind 0.0.0.0:$PORT`

## 5) Important free-tier note
- Current setup uses SQLite by default (free and simple).
- On free hosting, local disk can be ephemeral. Data may reset after redeploy/restart.
- If you need persistent production data, attach an external PostgreSQL service and set:
  - `POSTGRES_DB`
  - `POSTGRES_USER`
  - `POSTGRES_PASSWORD`
  - `POSTGRES_HOST`
  - `POSTGRES_PORT`

## 6) After deploy
- Open your Render URL.
- Register a user and verify pages: Home, Explore, Reels, Profile, Chat.

## 7) Optional frontend CDN optimization (still free)
- Keep templates on Django.
- Move media to an external object store only if needed for scale.
